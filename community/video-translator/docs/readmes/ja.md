# 🎬 リアルタイム動画字幕翻訳

[`lingo.dev`](https://lingo.dev/)SDKを使用してビデオ字幕をリアルタイムで翻訳するシステム。モノレポアーキテクチャで設計：フロントエンドはVercel、WebSocket翻訳サーバーはRenderで稼働。

## 課題

グローバル食品企業が国際市場への進出を目指しています。彼らのウェブサイトやビデオコンテンツ（UI文、SEOコンテンツ、食品関連動画を含む）は複数言語に対応する必要があります。

現在、すべての動画コンテンツは英語のみで、字幕を手動で翻訳するのは時間と費用がかかります。企業が求めるAI駆動型ソリューションは：

- ウェブサイトUIを複数言語に自動翻訳

- ビデオ字幕をリアルタイムで生成・翻訳

- SEOに最適化された多言語コンテンツの確保

目標は時間短縮、コスト削減、翻訳チームを拡大せずにシームレスな多言語体験の提供です。

## 始め方

1. リポジトリをクローン
```
git clone https://github.com/ShubhamOulkar/lingo.video.git
cd lingo.video
```
2. 依存関係をインストール
```
pnpm install
```
3. [`lingo.dev`](https://lingo.dev/)からAPIキーを取得
4. `.env`ファイルを作成し`LINGODOTDEV_API_KEY`を保存
5. フロントエンドとWebSocketサーバーを同時に実行
```
pnpm dev
```

## グローバル企業への影響とメリット

このシステムは組織、特にグローバル食品・配送企業に具体的なメリットを提供します：

- `VTTファイル管理の排除`：各言語の.vtt字幕ファイルを手動で作成・保存する必要なし。

- `データベースとストレージコストの削減`：字幕はその場で生成・翻訳されるため、複数言語ファイルの保存コストが不要。

- `開発者の作業負担軽減`：多言語ビデオコンテンツ維持に追加開発努力不要。

- `早期市場参入`：動画を数ヶ月ではなく数日で展開でき、グローバル展開を加速。

- `無制限の言語サポート`：AI駆動翻訳により世界中どの国へのリーチも可能。

- `翻訳ではなく製品に集中`：システムが自動的に多言語コンテンツを処理する間、チームは中核製品の改善に集中可能。

## 機能

- **リアルタイム字幕翻訳**  
  - [`lingo.dev`](https://lingo.dev/en/sdk) SDKとWebSocketサーバーを使用してビデオ字幕をその場で翻訳。  
  - 複数言語の`.vtt`ファイルを維持する必要なし。
  > 注：このリポジトリには手動精度テスト用の[.vttファイル](./apps/next-app/public/subtitles/emotions.hi.vtt)が含まれています。`CC`をクリックしてライブ翻訳と比較テスト可能。

- **ReactでのUI翻訳**  
  - [`Lingoコンパイラ`](https://lingo.dev/en/compiler)を使用してReact UIが自動更新 ⚡🤖。  
  - 翻訳をハードコードせずに動的言語コンパイル。  

- **SEOフレンドリーな多言語コンテンツ**  
  - [`Lingo CLI`](https://lingo.dev/en/cli)を使用してメタタグとOpen Graph（OG）タグを自動生成。  
  - CI/CDパイプラインで完全自動化可能。
  > 注：ヒンディー語のOGカードは[こちら](https://opengraph.dev/panel?url=https%3A%2F%2Flingo-video.vercel.app%2Fhi)で確認できます 

- **時間とコスト効率**  
  - 開発者の労力削減と第三者翻訳者の排除。  
  - 多言語コンテンツを**数ヶ月ではなく数日**で展開。  

- **無制限の言語サポート**  
  - AI駆動翻訳により世界中どの国へのリーチも可能。  
  - 手作業なしで新言語を簡単に追加。  

- **翻訳ではなく製品に集中**  
  - 翻訳が自動的に行われる間、チームは中核製品の改善に集中可能。  

- **ビデオ量に合わせて拡張**  
  - 追加インフラや保守なしで大量のビデオを処理可能。

- **ユーザー設定のシステムテーマに対応** 
  - ウェブサイトはユーザーが好むライトまたはダークテーマに自動的に対応可能。

## リアルタイム翻訳の課題と解決策

リアルタイム翻訳システムには複数の技術的・運用的課題があります。本プロジェクトは遅延を最小化し、翻訳コストを削減し、大量の動画コンテンツ全体で一貫した精度を確保するための本番環境グレードのソリューションを備えています。

### ⚠️ 主要な課題

1. **ネットワーク遅延** : リアルタイム翻訳には高速なWebSocket通信が必要です。ネットワークの不安定さが字幕更新の遅延を引き起こす可能性があります。

2. **LLMトークン生成の遅延** : 翻訳品質はLLMからのトークン生成速度に依存します。高負荷や大きな字幕は応答時間を増加させます。Lingo SDKはストリーミングをサポートしていません。

3. **冗長な翻訳コスト** : 多くの字幕は動画間で同じテキストを繰り返します。最適化がなければ、同じトークン生成に複数回課金されます。

4. **コールドスタートの問題** : サーバーレスデプロイメントは起動時間が遅くなり、リアルタイム字幕配信に影響を与える可能性があります。

5. **高トラフィックでのスケーリング** : 複数のユーザーが同時に動画を視聴すると、最適化されていない場合、翻訳またはソケットサーバーに過負荷がかかる可能性があります。
